# Website Type

Use this type for marketing sites, docs sites, public products, content-heavy experiences, and social-style interfaces.

## Shared Focus

- Strong page structure
- Clear routing
- Intentional design system
- Reusable content components

## Typical Folders

```text
project/
  app/
    pages/
    components/
    routes/
    assets/
  docs/
```

## Notes

- `website` may run in either `cs` or `sw`
- Social media style products can still be `website + sw`
- Keep content, design, and routing consistent across pages
